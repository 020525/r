算术运算

R中常用的算术运算符包括：

运算符 功能
+, -, *, /     加、减、乘、除
^或** 幂，例如：2^4的结果为16，2**0.5的结果是1.414214
%% 求余数，例如：5%%2的结果为1
%/% 求商（整数），例如：6.5%/%2.7的结果为2

以上运算符可以对标量进行运算，也可以对向量进行运算。向量运算的结果仍然是向量。

a <- c(1:10); b <- c(2, 5)
a/2
##  [1] 0.5 1.0 1.5 2.0 2.5 3.0 3.5 4.0 4.5 5.0
a/b  #分子依次取1,2,...,10；分母依次取2,5,2,5,...
##  [1] 0.5 0.4 1.5 0.8 2.5 1.2 3.5 1.6 4.5 2.0

集合运算

向量和列表被R当作集合使用，常用的集合运算包括：

函数 功能
union(x, y) 计算集合x和y的并集
intersect(x, y) 计算集合x和y的交集
setdiff(x, y) 求出只在x中但不在y中的元素集合
setequal(x, y) 判断集合x和y是否相同
is.element(x, y) 逐个判断x的每个元素是否在集合y中，返回一个与x等长的逻辑向量
x %in% y 功能与is.element(x, y)相同
match(x, y) 计算出x的每个元素在集合y中的位置

例如：

x <- c(1, 2, 3, 4);  y <- c(3, 4, 5); z <- c(2, 1, 3, 4)
union(x, y)  #求并集
## [1] 1 2 3 4 5
intersect(x, y)   #求交集
## [1] 3 4
setdiff(x, y)   #求集合差
## [1] 1 2
setequal(x, y)  #判断两个集合是否相同
## [1] FALSE
setequal(x, z)  #判断集合是否相同时，不考虑元素的顺序
## [1] TRUE
x==z   #对应比较x的每个元素与z的每个元素
## [1] FALSE FALSE  TRUE  TRUE
is.element(3, y)  #判断3是否在集合y中
## [1] TRUE
is.element(x, y)  #依次判断x的每个元素是否在y中
## [1] FALSE FALSE  TRUE  TRUE
x <- c(1, 2, 3, 4); z <- list(2, 3, "abc")
is.element(x, z)  #列表也可以被当作集合
## [1] FALSE  TRUE  TRUE FALSE
x <- c(1, 2, 3, 4);  y <- c(3, 4, 5)
match(5, y)  #查询5在集合y中的位置
## [1] 3
match(x, y)  #查询x的每个元素在集合y中的位置
## [1] NA NA  1  2

另一组常用的集合函数用于处理单个集合：

函数 功能
unique(x) 返回集合x的值域
duplicated(x) 依次判断集合x中的元素是否在之前重复过

例如：

x <- c(1, 2, 3, 2)
unique(x)   #计算x的值域，即去掉x中重复元素
## [1] 1 2 3
duplicated(x)  #依次判断集合x中的元素是否在之前重复过
## [1] FALSE FALSE FALSE  TRUE

逻辑运算

R中常用的逻辑运算包括：

运算符 功能
<, <=, >, >=    小于、小于等于、大于、大于等于。例如：3>2的结果是TRUE
==, != 等于、不等于。例如：3==2的结果是FALSE
!x 非x。例如：!3>2的结果是FALSE
x&y x且y，向量计算
x&&y x且y，标量计算
x|y x或y，向量计算
x||y x或y，标量计算
any(x) 如果逻辑向量x中存在有TRUE，那么返回TRUE；否则返回FALSE
all(x) 如果逻辑向量x全都是TRUE，那么返回TRUE；否则返回FALSE
which(x) 返回逻辑向量x中为TRUE的元素的下标

对于进行比较大小的计算，向量运算的结果仍然是向量。

mtcars$mpg
##  [1] 21.0 21.0 22.8 21.4 18.7 18.1 14.3 24.4 22.8 19.2 17.8 16.4 17.3 15.2
## [15] 10.4 10.4 14.7 32.4 30.4 33.9 21.5 15.5 15.2 13.3 19.2 27.3 26.0 30.4
## [29] 15.8 19.7 15.0 21.4
mtcars$mpg>20
##  [1]  TRUE  TRUE  TRUE  TRUE FALSE FALSE FALSE  TRUE  TRUE FALSE FALSE
## [12] FALSE FALSE FALSE FALSE FALSE FALSE  TRUE  TRUE  TRUE  TRUE FALSE
## [23] FALSE FALSE FALSE  TRUE  TRUE  TRUE FALSE FALSE FALSE  TRUE

x&y是对两个逻辑向量的元素两两对应进行AND运算，输出值也是向量。例如：

x <- c(TRUE, TRUE, FALSE)
y <- c(FALSE, TRUE, TRUE)
x&y
## [1] FALSE  TRUE FALSE

x&&y是对两个标量进行AND运算，计算结果也是一个标量。例如：

x <- TRUE
y <- FALSE
x && y
## [1] FALSE

注意，在x&&y计算中，如果x或y是向量，那么R会取出它们的第1个元素作为x或y的值，将向量当作标量使用，得到的结果仍然是标量。试比较：

x <- c(TRUE, TRUE, FALSE)
y <- c(FALSE, TRUE, TRUE)
x&y  #向量运算
## [1] FALSE  TRUE FALSE
x&&y #标量运算
## [1] FALSE

对于|和||也有类似的规则，|进行向量OR运算，||进行标量OR运算。

&&和||通常只用在if语句中使用，在其他场景通常是使用&和|

any(), all(), which()这三个函数常用来测试向量的内容。例如：

x <- c(TRUE, TRUE, FALSE, TRUE)
any(x)
## [1] TRUE
all(x)
## [1] FALSE
which(x)
## [1] 1 2 4

R在进行AND、OR运算时，会执行短路运算（与C语言相同），即在计算一个逻辑表达式的值时，如果根据表达式据前面的部分就已经可以确定最终的结果了，那么表达式后面的部分会被跳过。

if(3>0 || NA) print("OK")
## [1] "OK"

由于根据“3>0”就已经可以断定“3>0 || NA”的值必为TRUE，所以“NA”实际上并没有参与逻辑计算。

逻辑值可以参与数值计算，TRUE会被当作1，FALSE会被当作0。例如：

sum(mtcars$wt>2.0)   #计算重量超过2.0的汽车数量
## [1] 28
mean(mtcars$wt>2.0)  #计算重量超过2.0的汽车占比
## [1] 0.875

用逻辑向量进行筛选

要访问一个向量myvector的若干元素，可以使用myvector[v]，其中v包含了各元素的下标。例如：result <- myvector[c(1, 3, 4)]。

此外，我们也可以使用一个逻辑向量来进行筛选，语法是：

result <- myvector[logicVector]

其中，logicVector是一个由逻辑值组成的向量，长度与myvector相同。对于myvector中的每个元素：

  • 如果它在logicVector对应的元素为TRUE，那么它会被放入result；
  • 如果它在logicVector对应的元素为FALSE，那么它会被跳过；
  • 如果它在logicVector对应的元素为NA，那么会将NA放入result。
workmates <- c("Zhao", "Qian", "Sun", "Li")
isFriend <- c(TRUE, FALSE, TRUE, NA)
myfriend <- workmates[isFriend]
myfriend
## [1] "Zhao" "Sun"  NA

对数据框中的行和列，我们也可以用逻辑向量进行类似操作。

mydata
##   patientID    admDate age diabetes    status
## 1         1 10/15/2014  25    Type1      Poor
## 2         2 11/01/2014  34    Type2  Improved
## 3         3 10/21/2014  28    Type1 Excellent
## 4         4 10/28/2014  52    Type1      Poor
col.selector <- c(TRUE, FALSE, TRUE, FALSE, FALSE)
mydata[col.selector]    #用逻辑向量筛选列
##   patientID age
## 1         1  25
## 2         2  34
## 3         3  28
## 4         4  52
row.selector <- c(FALSE, FALSE,TRUE, TRUE)
mydata[row.selector,]   #用逻辑向量筛选行
##   patientID    admDate age diabetes    status
## 3         3 10/21/2014  28    Type1 Excellent
## 4         4 10/28/2014  52    Type1      Poor
mydata[mydata$age>30,]  #筛选出age>30的病人
##   patientID    admDate age diabetes   status
## 2         2 11/01/2014  34    Type2 Improved
## 4         4 10/28/2014  52    Type1     Poor

常见错误：

  • 漏写数据框的名字，例如mtcars[mpg>10, ]是错误的，正确的写法是mtcars[mtcars$mpg>10, ]
  • 漏写逗号，例如mtcars[mtcars$mpg>10]是错误的，正确的写法是mtcars[mtcars$mpg>10, ]
  • 使用连接不等式，例如cars[15<cars$speed<20,]是错误的，正确的写法是cars[cars$speed>15 & cars$speed<10, ]

缺失值

在R中，NA是一个特殊值，它是指Not Available，表示未知值或缺失值。几乎所有有NA参与的运算，其结果都是NA。

a <- c(10, 20, NA, 5)
a>=10
## [1]  TRUE  TRUE    NA FALSE
sum(a)   #求和
## [1] NA

向量的修改

(1)修改向量元素的值

可以直接为向量中的元素赋值，从而改变向量的值。

a <- 1:6
a[2] <- 20
a[3:5] <- c(30, 40, 50)
a
## [1]  1 20 30 40 50  6

(2)删除向量中的元素

在R中，向量都是常数，所以是不能编辑的（即不能删除或增加元素）。要删除一个向量中的元素，只能用剩下的元素来创建一个新向量，再赋值给原向量变量。

a <- 1:5
a <- a[c(1, 2, 4, 5)]  #删掉a[3]
a
## [1] 1 2 4 5

另外，向量的下标可以使用负数，负数的下标表示要剔除相应的元素。我们可以使用负数下标来删除向量中的元素。

a <- 1:5; a <- a[-1]; a    #等价于a[c(2,3,4,5)]
## [1] 2 3 4 5
a <- 1:5; a <- a[-1:-3]; a  #即a[c(-1, -2, -3)]
## [1] 4 5

注意，正值下标与负值下标不能混合在一起。例如，a[c(-1,3)]是错误的表达式。

(3)插入元素

方法是创建一个新的向量，包含原元素和插入的元素，再将新向量赋值给原向量变量。

a <- 1:6
a <- c(a[1:3], 10, 20, 30, a[4:6])   #插入10, 20, 30
a
## [1]  1  2  3 10 20 30  4  5  6

(4)添加元素

为向量添加新的元素，只需要将元素赋值给指定的下标即可。

a <- c(1, 3, 10);  a[4] <- 20;  a   #追加一个元素
## [1]  1  3 10 20
a <- c(1, 3, 10);  a[6] <- 30;  a   #添加元素时，允许跳跃
## [1]  1  3 10 NA NA 30

追加元素是一种常见的操作，除了可以用上面的方法外，还可以：

a <- c(1, 3, 10);  a <- c(a, 20); a 
## [1]  1  3 10 20
a <- c(1, 3, 10);  a <- c(a, 20, 30, 50); a 
## [1]  1  3 10 20 30 50

以上对向量的修改操作对因子同样适用。需要注意的是，因子的元素只能在其levels范围内取值。例如：

diabetes <-c("Type1","Type2","Type1")
diabetes <- factor(diabetes, ordered=FALSE, levels=c("Type1", "Type2"))
diabetes
## [1] Type1 Type2 Type1
## Levels: Type1 Type2
diabetes[1] <- "Type2"   #修改元素
diabetes[4] <- "Type1"   #添加一个元素
diabetes
## [1] Type2 Type2 Type1 Type1
## Levels: Type1 Type2

循环补齐规则

R中的许多计算（例如sqrt、加减乘除等）都是向量化的，即支持对向量进行运算。对于这些运算，通常要求参与运算的向量具有相同的长度，如果向量的长度不相等，那么R会将短的向量重复，使得它与长的向量等长，这个规则称为Recycling Rule。

例如：计算c(1,3,4,5) + c(10,20)，R会将c(10, 20)拉长为c(10, 20, 10, 20)，这样参与计算的两个向量就等长了，之后两个向量的元素对应相加，结果为c(11, 23, 14, 25)。

循环补齐规则被R广泛使用在各种计算中。例如：计算mtcars$wt > 2.0时，R会将2.0拉长为c(2.0, 2.0, ..., 2.0)，之后再与mtcars$wt比较，计算的结果是一个向量。

注意，循环补齐时，要求短向量被重复整数倍，即长向量的长度是短向量长度的整数倍。如果短向量不是被重复了整数倍，那么有些函数会给出一个警告，但仍可运行，而另一些函数不接受这种情况，会报错。

c(1:5)+c(10,20,30)  #会报出一个warning，仍可得到一个结果

data.frame(id=c("a", "b"), age=c(19, 18, 17, 20)) #短向量被重复整数倍，OK

data.frame(id=c("a", "b"), age=c(19, 18, 17))  #会报出一个error，无结果

创建新的列变量

在使用函数edit()编辑数据框时可以增加列变量，并输入值。对于一个给定的数据框，我们还可以写代码增加新的列变量。

为数据框mydata增加一列var的语法是：

mydata$var <- vectorValue

例如：数据框cars记录了50次刹车实验的结果，共有两列，speed列记录了刹车时的速度（单位：英里/小时），dist列记录了刹车所用的距离（单位：英尺）。现在，我们计算刹车所用的时间，保存为新的一列time。换算公式为：

\[ time = \frac{dist}{5280*speed/2}*3600 \]

#为数据框cars增加一列。1 mile = 5280 feet.
cars$time <- cars$dist/(5280*cars$speed/2)*3600

列变量的重命名

为一个数据框的列重命名，可以使用edit()打开编辑器，在编辑器中修改。另一种方法是使用函数names()，语法为：

names(mydata) <- colnames

其中，colnames是一个字符串向量，包含了各列的名字。

names(cars)
## [1] "speed"     "dist"      "speedtype"
names(cars) <- c("速度", "距离", "类型")
names(cars)
## [1] "速度" "距离" "类型"

names(mydata)返回一个向量，包含数据框mydata所有列的名字，我们可以只从这个向量中取出若干个，只为这些列重命名。

names(cars)
## [1] "速度" "距离" "类型"
names(cars)[c(1,3)] <- c("speed", "speedtype")
names(cars)
## [1] "speed"     "距离"      "speedtype"

处理缺失值

在实际数据框中，出现缺失值（NA）几乎是不可避免的。由于NA参与几乎所有的计算都会导致计算结果为NA，因此在分析时通常需要把缺失值排除掉。

在许多数值函数中，都有一个na.rm参数，设置na.rm=TRUE将在计算时排除NA。例如：

x <- c(10, 20, NA, 3)
y <- sum(x); y
## [1] NA
z <- sum(x, na.rm=TRUE); z
## [1] 33

我们也可以使用函数na.omit()，将数据框中包含缺失值的行都剔除掉，语法为：

newdata <- na.omit(mydata)

注意，na.omit()将把含有NA的行删除掉，即只要一行中任意列有NA，这一行就会被删掉。例如：

studentID <- c(201101, 201103, 201201, 201202)
studentName <- c("Zhao", "Qian", "Sun", "Li")
English <- c(90, NA, 80, 85)
Physics <- c(85, 70, NA, 80)
mydata <- data.frame(studentID, studentName, English, Physics,stringsAsFactors=FALSE)
mydata
##   studentID studentName English Physics
## 1    201101        Zhao      90      85
## 2    201103        Qian      NA      70
## 3    201201         Sun      80      NA
## 4    201202          Li      85      80
newdata <- na.omit(mydata)
newdata
##   studentID studentName English Physics
## 1    201101        Zhao      90      85
## 4    201202          Li      85      80

由于na.omit()过滤掉数据太多，实际并不常用。通常的做法是，根据计算的需要，按列来过滤掉NA数据。

studentID <- c(201101, 201103, 201201, 201202)
studentName <- c("Zhao", "Qian", "Sun", "Li")
English <- c(90, NA, 80, 85)
Physics <- c(85, 70, NA, 80)
mydata <- data.frame(studentID, studentName, English, Physics,stringsAsFactors=FALSE)

#要分析英语成绩时，只需要根据English列来过滤数据
newdata <- mydata[!is.na(mydata$English),]
newdata
##   studentID studentName English Physics
## 1    201101        Zhao      90      85
## 3    201201         Sun      80      NA
## 4    201202          Li      85      80
summary(newdata$English)
##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
##    80.0    82.5    85.0    85.0    87.5    90.0

处理日期值

日期型数据也是一种衍生数据，这种数据不能直接输入，通常是先输入字符串表示的日期，然后再将字符串转换为日期。

将字符串转换为日期的函数是as.Date()，语法为：

mydate <- as.Date(str, format=FormatString)

其中，format参数用来指定字符串str的格式。例如：

mydate <- as.Date("2018年03月06日", format="%Y年%m月%d日")
class(mydate)

%A 星期名，例如Monday、星期一

自动数据类型转换

对于R的许多系统函数，例如c()，在必要时，这些函数会自动进行数据类型转换。总的转换顺序是：将logical转换为integer(TRUE当作1L，FASLE当作0L)，将integer转换为double/numeric，将double/numeric转换为character。例如：

## [1]  1 10

## [1] "double"

a <- c(1, "a", 3)    #自动将double转换为character，等价于 a <- c("1", "a", "3")
a
## [1] "1" "a" "3"

数据排序

对一个向量（数值型、字符型）排序的函数是：

myresult <- order(myvector, decreasing=FALSE, na.last=TRUE)

函数order()返回排序后各位置上的下标编号。decreasing=FALSE表示进行升序排列（默认），decreasing=TRUE表示进行降序排列。na.last=TRUE表示将NA值排在最后（默认），na.last=FALSE表示将NA值排在最前。

a <- c(10, 9, 11, 7)

order(a)   #第4个元素最小，第2个元素次小，第1个元素更大一点，第3个元素最大
## [1] 4 2 1 3

order(-a)   #等价于 order(a, decreasing=TRUE)
## [1] 3 1 2 4

因此，将一个向量排序的方法是：

newvector <- myvector[order(myvector)]

例如：

a <- c(10, 9, 11, NA, 7)

a[order(a)]   #先用order()计算出下标，再依次取出元素
## [1]  7  9 10 11 NA

a[ order(a, na.last=FALSE) ]  #把NA值放在最前面
## [1] NA  7  9 10 11

Do It Yourself

对向量进行排序是非常常用的操作，如果只想要排序后的结果，可以使用函数sort()。

a <- c(10, 9, 11, 7)

a[order(a, decreasing=TRUE)]
## [1] 11 10  9  7

sort(a, decreasing=TRUE)  #直接输出排序结果。该函数不能进行多重排序
## [1] 11 10  9  7

函数rank(x)可计算出各元素在排序后的名次，默认为升序排序。

a <- c(10, 9, 11, 7)
rank(a)   #升序排序，10排第3位，9排第2位，11排第4位，7排第1位（最小）
## [1] 3 2 4 1

a[rank(a)<=2]  #找出a最小的2个元素
## [1] 9 7

a <- c(10, 9, 11, 10)   #如果有相等的元素，那么相等的元素构成一个tie
rank(a, ties.method="min")   #一个tie的各元素的序号由ties.method决定
## [1] 2 1 4 2

#tie.method可取"average", "first", "last", "random", "max", "min"之一

数据集的合并

把多个行数相同的数据框合并（即进行列合并）的方法是：

newdata <- cbind(data1, data2, …)

cbind()在合并时只要求data1, data2, ...具有相同的行数，在合并时，只是简单地把各列拼接起来。

workerID <- c(1:4)
workerAge <- c(22, 25, 20, 21)
data1 <- data.frame(workerID, workerAge)

workerName <- c("Zhao", "Qian", "Sun", "Li")
workerGender <- c("Male", "Male", "Female", "Male")
data2 <- data.frame(workerName, workerGender)

mydata <- cbind(data1, data2)
mydata
##   workerID workerAge workerName workerGender
## 1        1        22       Zhao         Male
## 2        2        25       Qian         Male
## 3        3        20        Sun       Female
## 4        4        21         Li         Male

可见，在用cbind()合并时，各数据框中行的顺序要一致。

更安全的列合并方式是使用函数merge()，这个函数在合并时使用一个关键字列，语法是：

newdata <- merge(data1, data2, by=KeyCol)

其中，KeyCol是data1和data2都有的列，以该列为合并的依据。data1和data2中行的顺序可以不一致。

workerID <- c(1:4)
workerAge <- c(22, 25, 20, 21)
data1 <- data.frame(workerID, workerAge)

workerID <- c(4:1)
workerName <- c("Zhao", "Qian", "Sun", "Li")
workerGender <- c("Male", "Male", "Female", "Male")
data2 <- data.frame(workerID,workerName, workerGender)

mydata <- merge(data1, data2, by="workerID")
mydata
##   workerID workerAge workerName workerGender
## 1        1        22         Li         Male
## 2        2        25        Sun       Female
## 3        3        20       Qian         Male
## 4        4        21       Zhao         Male

另外，merge()还可以合并不完全对齐的数据，语法是：

newdata <- merge(data1, data2, by.x=key1, by.y=key2, all.x | all.y | all)

其中，key1是data1的列，key2是data2的列，在合并时将使用这两个列的数据作为关键字；all.x、all.y、all可任选一个设置为TRUE。

merge()会首先计算出data1和data2的笛卡尔积，然后根据all.x、all.y或all的值从笛卡尔积中挑选出一部分数据：

  • all.x=TRUE，表示data1的行将全部保留下来，这种操作也称为left join；
  • all.y=TRUE，表示data2的行将全部保留下来，这种操作也称为right join；
  • all=TRUE，表示两个数据框的数据都将被保留；
  • 如果以上三个参数都未使用，那么只会保留在data1和data2都出现的行。
workerName <- c("a", "b", "c", "d")
workerAge <- c(23, 22, 22, 25)
data1 <- data.frame(workerName, workerAge)

anotherName <- c("c", "d", "e")
anotherSalary <- c(2200, 2500, 2000)
data2 <- data.frame(anotherName, anotherSalary)

merge(data1, data2, by.x="workerName", by.y="anotherName", all.x=TRUE)
##   workerName workerAge anotherSalary
## 1          a        23            NA
## 2          b        22            NA
## 3          c        22          2200
## 4          d        25          2500
merge(data1, data2, by.x="workerName", by.y="anotherName", all.y=TRUE)
##   workerName workerAge anotherSalary
## 1          c        22          2200
## 2          d        25          2500
## 3          e        NA          2000
merge(data1, data2, by.x="workerName", by.y="anotherName", all=TRUE)
##   workerName workerAge anotherSalary
## 1          a        23            NA
## 2          b        22            NA
## 3          c        22          2200
## 4          d        25          2500
## 5          e        NA          2000
merge(data1, data2, by.x="workerName", by.y="anotherName")
##   workerName workerAge anotherSalary
## 1          c        22          2200
## 2          d        25          2500

注意：merge()在合并两个数据框时，会将by.x和by.y合并成一列，在合并结果中这一列被命名为by.x的值。请观察上面的例子。

使用函数merge()进行左连接（或右连接）常用于将分散的多个小数据集整合成一个完整的大数据集。例如：

student <- data.frame(
  studentName=c("Liu", "Zhang", "Cheng", "Wang", "Li"),
  home=c("WH", "BJ", "WH", "SH", "WH"),    #学生的户籍地（简称）
  stringsAsFactors = FALSE
)

city <- data.frame(
  cityName=c("WH", "BJ", "SH", "GZ"),
  fullName=c("Wuhan", "Beijing", "Shanghai", "Guangzhou"),
  stringsAsFactors = FALSE
)

#将每个学生的户籍地由简称翻译成全称
student2 <- merge(student, city, by.x="home", by.y="cityName", all.x=TRUE)
student2
##   home studentName fullName
## 1   BJ       Zhang  Beijing
## 2   SH        Wang Shanghai
## 3   WH         Liu    Wuhan
## 4   WH       Cheng    Wuhan
## 5   WH          Li    Wuhan

将两个具有相同列的数据框合并（即进行行合并）的方法是：

newdata <- rbind(data1, data2, …)

其中，data1、data2、...的列结构相同，即列数相同，且各列的名字也相同。

workerID <- c(1:4)
workerName <- c("Zhao", "Qian", "Sun", "Li")
data1 <- data.frame(workerID, workerName)

workerID <- c(5:7)
workerName <- c("Zhou", "Wu", "Zheng")
data2 <- data.frame(workerID, workerName)

mydata <- rbind(data1, data2)
mydata
##   workerID workerName
## 1        1       Zhao
## 2        2       Qian
## 3        3        Sun
## 4        4         Li
## 5        5       Zhou
## 6        6         Wu
## 7        7      Zheng

随机抽样

在正式分析大数据集之前，通常需要先抽取一个小样本进行实验。随机抽样会用到三个函数：nrow()、sample()和set.seed()。函数nrow()返回数据框的行数，语法为：

n <- nrow(mydata)

函数sample()用于从一个向量中随机抽取出若干个元素，语法为：

myset <- sample(vector, n, replace=FALSE)

该函数会从向量vector中随机抽取出n个元素，参数replace=TRUE表示进行放回抽样，replace=FALSE表示进行不放回抽样，默认进行不放回抽样。

nrow(mtcars)  #数据框mtcars共有32行
## [1] 32
sample(1:32, 5)
## [1] 19 20 21 29  8
sample(1:32, 5)
## [1] 30  3  5 19 20
a <- sample(1: nrow(mtcars), 5)
mydata <- mtcars[a, ]  #从mtcars中随机抽取5行
mydata
##                   mpg cyl  disp  hp drat    wt  qsec vs am gear carb
## Pontiac Firebird 19.2   8 400.0 175 3.08 3.845 17.05  0  0    3    2
## Mazda RX4 Wag    21.0   6 160.0 110 3.90 2.875 17.02  0  1    4    4
## Merc 280C        17.8   6 167.6 123 3.92 3.440 18.90  1  0    4    4
## Hornet 4 Drive   21.4   6 258.0 110 3.08 3.215 19.44  1  0    3    1
## Duster 360       14.3   8 360.0 245 3.21 3.570 15.84  0  0    3    4

函数sample()进行抽样时，在内部会用到随机数。

计算机上得到的随机数是一种根据算法计算出的数值序列，并不是真正的随机数，因此称为伪随机数。这种伪数据数序列是由算法和初始种子来决定的，因此我们可以通过指定初始种子，来决定随机数序列。

设置随机数种子的函数是：

set.seed(n)

通过设置随机数种子，我们可以让sample()的抽样结果可重现(reproducible)。

set.seed(23)
mydata <- mtcars[sample(1: nrow(mtcars), 5), ]  #从mtcars中随机抽取5行
mydata
